• Arsenal: distributed SFTP based file system (2010)
  - My solution to redundant and distributed personal network storage. It
    allows any computer on your network to act as a dumb storage node. Any
    clients can use Arsenal to manage a battery of SSH servers. The Arsenal
    clients can be configured with the network topology with a simple XML file.
  - Currently Arsenal operates in read-only mode (so there is no way to lose
    data!)
  - One of the primary goals of the project (besides providing consolidated
    network storage) is to provide a solid and robust interface to your data.
    If Arsenal crashes your data will still be laid out logically on all disk
    drives so manual recovery is possible.

• StressChill/WeTap: Participatory Sensing with Android (2009-2010)

• loomlib: parallel data structures library (2009)
  - Implemented several data structures for simplifying the development of
    multi-threaded applications.
  - Asynchronous queue/list.
  - Thread pool. Essentially an asynchronous queue of function pointers.
  - Pipeline. Designate different pipeline stages (or pumps) that all run
    concurrently to process data in a stream-like fashion.
  - Tree. A pipeline with branches where each node may have multiple children.
    Problems where a data flows is easily split and never joined work well with
    this paradigm.

• Gumstix Nest Box Deployment  (2006-2007)
  - Deployed low powered audio recording devices and networked cameras into
    several nest boxes to monitor the nesting cycle of various bird species.
  - A detection and classification system was developed to determine the
    current nesting stage.
  - Adaptive sampling techniques were used to further reduce power consumption
    without significantly degrading classification.
• Cyclops Pitfall Traps (2006)
  - Built and deployed a network of low-powered wireless cameras into an array
    of pitfall traps.
  - One image was collected from each node every five minutes and aggregated on
    a nearby server for analysis.
  - A detector was implemented to raise alerts when objects fell into the trap.
  - Each node contained Cyclops and Mica2 boards and were programmed with
    EmStar and TinyOS.
